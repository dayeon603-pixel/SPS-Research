# Dev Log — 2026-03-24: WordNet Vocab Loading Memory Spike

## What Happened
`EmbeddingPerturbationFamily` tried to build synonym directions from WordNet for the entire tokenizer vocabulary (50k+ tokens) at import time. Massive memory spike, and silently fell back to random orthogonal directions for most tokens.

So the "synonym directions" were mostly random. The perturbations were not actually semantically grounded.

## Fix
- Added `vocab_size` cap (default 10k)
- Explicit logging when falling back to random directions
- `max_synonyms_per_token` parameter to control memory
- Lazy loading instead of import-time construction

## Impact
This was a silent correctness bug. Results computed before this fix may have used random directions instead of synonym-based perturbations. Need to re-run experiments from before this date.
