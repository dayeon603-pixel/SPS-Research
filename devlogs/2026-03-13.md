**Notation audit: $\varepsilon$ subscript on $\mathrm{Sens}$.**

The sensitivity functional is defined as $\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x)$ — it carries both a subscript for the transformation family $\mathcal{T}$ and a subscript for the perturbation radius $\varepsilon$. Both are essential: the functional depends on which family you're using AND what radius you're measuring within.

Going through the drafts:

- Definition 1: $\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x)$ ✓
- Definition 2: $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ ✓
- Theorem 1: $\mathrm{Sens}_{\mathcal{T}_1,\varepsilon}$ and $\mathrm{Sens}_{\mathcal{T}_2,\varepsilon}$ ✓
- Theorem 2: $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ ✓
- Theorem 3: **$\mathrm{Sens}_{\mathcal{T}}$ — WRONG, missing $\varepsilon$**
- Theorem 4: **$\mathrm{Sens}_{\mathcal{T}}$ — WRONG in three places, missing $\varepsilon$**
- Corollary 1: $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ ✓

Two theorems dropped the $\varepsilon$ subscript. This isn't just a typo. Writing $\mathrm{Sens}_\mathcal{T}$ (without $\varepsilon$) implies a global sensitivity functional — taking the supremum over all transformation radii, not just within $\varepsilon$. That's a different and strictly larger quantity:

$$\mathrm{Sens}_\mathcal{T}(f; x) = \sup_{T \in \mathcal{T}, c(T,x) > 0} \frac{d_\mathcal{Y}(f(x), f(Tx))}{c(T,x)}$$

vs.

$$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) = \sup_{T \in \mathcal{T}, 0 < c(T,x) \leq \varepsilon} \frac{d_\mathcal{Y}(f(x), f(Tx))}{c(T,x)}$$

The $\varepsilon$ version is local — it only considers transformations within the $\varepsilon$-ball. The non-$\varepsilon$ version considers all transformations regardless of magnitude. These coincide only when the sensitivity function is monotone in $c(T, x)$, which is not guaranteed.

Using $\mathrm{Sens}_\mathcal{T}$ in Theorems 3 and 4 overstates what the theorem actually proves. The Lipschitz bound in Theorem 3 bounds the sensitivity for any transformation (Lipschitz applies globally), so the non-$\varepsilon$ version would also hold — but that's a stronger claim than what's needed, and it's inconsistent with how Definition 1 is stated. Similarly, Theorem 4 is specifically about local sensitivity (the chain constraint requires both steps to be within $\varepsilon$) — using the global notation contradicts the hypothesis.

### Lab Notes

The $\varepsilon$ subscript drop is the kind of error that happens when you write proofs quickly without going back to the definition. In informal reasoning, "the sensitivity" is unambiguous because there's usually a fixed $\varepsilon$ in context. In formal proofs, every symbol needs its full label every time, or the statement becomes ambiguous.

For Theorem 3 specifically: the Lipschitz bound actually does hold for the global (non-$\varepsilon$) sensitivity — if $f$ is $L$-Lipschitz globally, then the normalized output change under any transformation is bounded by $L$. So the global version of the claim is also true. But stating it that way is a different theorem than what Definition 1 defines. The paper is about $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ specifically.

The lesson: when you define a parameterized family of objects (indexed by $\varepsilon$), carry the parameter in every statement. Don't let notation shortcuts smuggle in a change of meaning.

Also noticed: the notation table I added to `theory/definitions.md` should have been there from day one. Going forward, every new symbol gets added to the table before it's used in a theorem. "Define before use" is a code principle that applies equally to mathematical notation.
