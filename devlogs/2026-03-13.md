## [11:30 UTC] Whiteboard Session — $\varepsilon$ Subscript Audit and Meaning

### State at Session Start

Notation error found. But before doing a mechanical find-replace, I want to think carefully about whether the error is semantic (wrong claim) or presentational (correct claim, wrong notation).

### Work Completed

**Comparing the two versions:**

$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x)$: supremum over transformations with $c(T, x) \leq \varepsilon$. Local.

$\mathrm{Sens}_{\mathcal{T}}(f; x)$: supremum over all transformations regardless of magnitude. Global.

$$\mathrm{Sens}_{\mathcal{T}}(f; x) = \sup_{T \in \mathcal{T}, c(T,x) > 0} \frac{d_\mathcal{Y}(f(x), f(Tx))}{c(T,x)} \geq \mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x)$$

for any $\varepsilon > 0$. The global version is at least as large.

**Does the error in Theorem 3 matter?**

Theorem 3 says: if $f$ is $L$-Lipschitz, then $\mathrm{Sens}_{\mathcal{T}} \leq L$ (global, no $\varepsilon$).

This is actually true! The Lipschitz condition bounds $d_\mathcal{Y}(f(x), f(Tx)) \leq L \cdot c(T,x)$ for all $T$, not just those with $c \leq \varepsilon$. So $\mathrm{Sens}_{\mathcal{T}}(f; x) \leq L$ holds globally.

But: the SPS definition uses $\mathrm{Sens}_{\mathcal{T},\varepsilon}$, not $\mathrm{Sens}_{\mathcal{T}}$. If I use the global sensitivity in Theorem 3's bound but the local sensitivity in the SPS definition, the result is internally inconsistent — I'm proving a bound on a quantity that's different from what appears in SPS.

The correct statement: "$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) \leq L$" — the local sensitivity is also bounded by $L$ since $\mathrm{Sens}_{\mathcal{T},\varepsilon} \leq \mathrm{Sens}_{\mathcal{T}} \leq L$. So the Lipschitz bound applies to the local version too. The $\varepsilon$ version of Theorem 3 is: $\mathrm{SPS}_\varepsilon \geq e^{-L}$.

**Does the error in Theorem 4 matter?**

Theorem 4 uses $\mathrm{Sens}_{\mathcal{T}}$ (global) on the RHS of the chain bound. But the chain constraint says $c(T_1, x) \leq \varepsilon$ and $c(T_2, T_1 x) \leq \varepsilon$. The sensitivity at $x$ is evaluated within radius $\varepsilon$ — so the local version $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ is what should appear. Using the global version $\mathrm{Sens}_{\mathcal{T}}$ makes the RHS bound larger (global $\geq$ local), so the inequality still holds. But it's a looser bound than what the proof actually establishes.

The corrected version uses $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ throughout — tighter, and consistent with the definitions.

**Verdict:** The error matters semantically. Both theorems are stating weaker-than-possible claims by using the global sensitivity where the local one should appear. The proofs are correct with either version, but the paper should state the tighter, locally-defined version.

**Counter-example to motivate the distinction:**

Consider a model $f$ that is $L$-Lipschitz globally but has $\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) = 0.01$ (very small local sensitivity). If we write the SPS bound using $\mathrm{Sens}_\mathcal{T}$ instead of $\mathrm{Sens}_{\mathcal{T},\varepsilon}$, we get a bound of $e^{-L}$ instead of the much tighter $e^{-0.01}$. The difference between $e^{-L}$ and $e^{-0.01}$ can be enormous (for $L = 10$: $4.5 \times 10^{-5}$ vs $0.990$).

This is not just a notation issue — it's the key reason the local sensitivity framework is useful: local sensitivity can be much smaller than global Lipschitz, giving much tighter bounds.

### Results / Observations

Notation error is semantic, not just presentational. Writing $\mathrm{Sens}_{\mathcal{T}}$ vs $\mathrm{Sens}_{\mathcal{T},\varepsilon}$ implies different and possibly vacuous bounds. The local version is both correct and necessary for the bounds to be non-trivial.

### Hypotheses

1. The distinction between local and global sensitivity is the fundamental contribution of the $\varepsilon$-parameterized framework. Losing the $\varepsilon$ subscript loses the key feature.
