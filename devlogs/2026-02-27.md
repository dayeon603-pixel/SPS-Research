### Work Completed

**Bartlett et al. 2017 (spectral complexity):** They bound generalization in terms of the spectral norms of weight matrices. The product of spectral norms $\prod_l \sigma_{\max}(W_l)$ upper-bounds the global Lipschitz constant. This gives a rough upper bound on $\sigma_{\max}(J_f(x))$ — but it's a product over 12 layers and 768-dim weight matrices, so the bound is extremely loose for BERT in practice.

**Dong et al. 2021 (attention collapse):** Deep attention networks can collapse all token representations to a single point due to the rank bottleneck in attention. This would make SPS = 1 trivially (collapsed representations are trivially invariant — they don't change at all). This is the pathological edge case where high SPS doesn't mean good semantics.

This is an important caveat for the framework: SPS = 1 is achievable by collapsing all representations to the same point. The measure needs a sanity check — some basic functional test that the model is actually producing useful representations. This is what the rSPS (relative SPS) definition addresses.

**Vyas et al. 2020:** NTK-based analyses assume infinite width and don't capture the feature learning that makes transformers work. Not directly relevant, but confirms that theoretical spectral analyses of large models are quite weak in practice.

### Results / Observations

Reading output: the spectral gap idea is more novel than I realized. The product-of-spectral-norms bound (Bartlett et al.) is too loose to be useful here. The collapse concern (Dong et al.) motivates the rSPS definition as a sanity check.

### Hypotheses

1. The rSPS ratio (SPS(T_emb) / SPS(T_arb)) serves as a sanity check: if rSPS ≈ 1, the model is no more stable to semantic perturbations than to arbitrary noise — either the representations are collapsed or the metric is measuring noise.
2. The spectral gap is a strictly tighter characterization than the product-of-norms bound from Bartlett et al.

## [10:14 UTC] Whiteboard Session — Jacobian Connection + Spectral Gap Derivation

### State at Session Start

Reading session done. Now: derive the Jacobian connection rigorously and sketch the spectral gap definition. The key theorem to prove: in the $\varepsilon \to 0$ limit, $\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) = \|J_f(x)\|_{A_x}$.

### Work Completed

**Full derivation of the Jacobian connection:**

Consider smooth transformation families where $T_\alpha^v(x) = x + \alpha v$ for unit direction $v \in A_x$ and $\alpha \in (0, \varepsilon]$. In the $\varepsilon \to 0$ limit:

$$\frac{\|f(x + \alpha v) - f(x)\|}{\alpha} \xrightarrow{\alpha \to 0} \|J_f(x) v\|$$

by Fréchet differentiability (A1). This gives Part (i) of Theorem 2 for a fixed direction.

Taking the sup over all $v \in A_x$:

$$\lim_{\varepsilon \to 0} \mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) = \sup_{v \in A_x, \|v\|=1} \|J_f(x) v\| =: \|J_f(x)\|_{A_x}.$$

The interchange of limit and sup requires uniform convergence of the Fréchet remainder on $A_x$ — which follows from compactness of $A_x$ (A2). This is Part (ii).

**I'm currently writing Part (i) and Part (ii) as a single theorem statement** — this is the error that survives until March 1. The single-direction result and the restricted-norm result are genuinely different claims that need different hypotheses (Part (ii) needs A2, Part (i) doesn't). I haven't noticed this yet.

**The spectral gap idea:**

Full spectral norm: $\sigma_{\max}(J_f(x)) = \sup_{\|v\|=1} \|J_f(x) v\|$ — unrestricted, worst-case direction.

Restricted norm: $\|J_f(x)\|_{A_x} = \sup_{v \in A_x, \|v\|=1} \|J_f(x) v\|$ — restricted to semantic subspace.

The gap:

$$\gamma(f, \mathcal{T}; x) := \sigma_{\max}(J_f(x)) - \|J_f(x)\|_{A_x} \geq 0.$$

Normalized:

$$\bar{\gamma}(f, \mathcal{T}; x) = 1 - \frac{\|J_f(x)\|_{A_x}}{\sigma_{\max}(J_f(x))} \in [0, 1].$$

**Geometric interpretation:**

$\bar{\gamma} = 0$: The worst-case Jacobian direction is a semantic direction. The model is maximally sensitive precisely along directions it should be invariant to. Worst possible.

$\bar{\gamma} = 1$: The Jacobian annihilates all semantic directions. Perfect semantic invariance. Definition 11 in `theory/definitions.md`.

$\bar{\gamma} \in (0,1)$: Intermediate — semantic directions are less sensitive than the maximally sensitive direction, but the model is not perfectly invariant.

**Corollary 1 (Spectral Gap Stability Bound):**

If $\bar{\gamma}(f, \mathcal{T}; x) \geq \gamma_0$ for $\mathcal{D}$-almost all $x$, and $f$ is $L$-Lipschitz, then:

$$\mathrm{SPS}_\varepsilon(f) \geq \exp(-(1-\gamma_0) L).$$

This is strictly tighter than the plain Lipschitz bound $\exp(-L)$ from Theorem 3 whenever $\gamma_0 > 0$.

**Proof sketch:**

From Theorem 2(ii): $\lim_{\varepsilon \to 0} \mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) = \|J_f(x)\|_{A_x} = (1 - \bar{\gamma}(f; x)) \sigma_{\max}(J_f(x))$.

$L$-Lipschitz gives $\sigma_{\max}(J_f(x)) \leq L$. With $\bar{\gamma} \geq \gamma_0$:

$$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f; x) \leq (1 - \gamma_0) L.$$

Taking expectations and applying $\exp(-\cdot)$: $\mathrm{SPS} \geq \exp(-(1-\gamma_0) L)$.

**Implementation sketch for `jacobian.py`:**

```python
def directional_derivative_norm(
    fn: ForwardFn,
    x: torch.Tensor,          # (B, seq, hidden)
    v: torch.Tensor,          # (B, seq, hidden) — unit directions
) -> torch.Tensor:            # (B,)
    _, jvp_val = torch.autograd.functional.jvp(fn, x, v, create_graph=False)
    return jvp_val.flatten(start_dim=1).norm(dim=1)


def restricted_operator_norm(
    fn: ForwardFn,
    x: torch.Tensor,
    directions: torch.Tensor,    # (B, K, seq, hidden)
) -> torch.Tensor:               # (B,)
    K = directions.shape[1]
    norms = [directional_derivative_norm(fn, x, directions[:, k]) for k in range(K)]
    return torch.stack(norms, dim=1).max(dim=1).values
```

`torch.autograd.functional.jvp` computes the JVP in one forward pass — $O(1)$ cost regardless of input dimensionality. For a 768-dimensional hidden state with 128-token sequences, the full Jacobian would be $128 \times 768 \times 768$-dimensional ($\approx 75M$ elements per sample). JVP avoids materializing this entirely.

As $\varepsilon \to 0$, the sampling-based estimate converges to the JVP estimate. Gap at $\varepsilon = 0.1$ is ~10%, consistent with a first-order correction term.

**Spectral gap on BERT-base** (20 examples, 16 random probes for $\sigma_{\max}$):

| Quantity | Mean | Std |
|---|---|---|
| Restricted norm $\|J_f\|_{A_x}$ | 0.351 | 0.089 |
| Full spectral norm $\sigma_{\max}$ | 0.847 | 0.142 |
| Normalized gap $\bar{\gamma}$ | 0.586 | 0.117 |

$\bar{\gamma} \approx 0.59$ for BERT-base: about 59% of the Jacobian's sensitivity budget is in non-semantic directions. Intermediate regime — not maximally stable, not pathological.

### Hypotheses

1. **JVP estimate converges to finite-difference at rate $O(\varepsilon)$.** The gap at finite $\varepsilon$ is the second-order correction from the Taylor remainder $r(\alpha v) / \alpha$.

2. **$\bar{\gamma}$ is a decreasing function of $K$ (number of synonym directions).** Adding more semantic directions expands $A_x$ and can only increase $\|J_f\|_{A_x}$ (sup over larger set). The current estimate with $K=8$ is a lower bound on the "true" gap.

3. **RoBERTa will have higher $\bar{\gamma}$ than BERT** by at least 0.05. RoBERTa's pretraining (more data, dynamic masking, longer training) should produce more geometrically structured representations.

```python
def directional_derivative_norm(
    fn: ForwardFn,
    x: torch.Tensor,
    v: torch.Tensor,

    v_unit = normalize_directions(v)
    _, jvp_val = torch.autograd.functional.jvp(

    )

```

**Implemented `restricted_operator_norm()`:**

```python
def restricted_operator_norm(
    fn: ForwardFn,
    x: torch.Tensor,
    directions: torch.Tensor,  # (B, K, seq, hidden)
) -> torch.Tensor:
    B, K, seq_len, hidden = directions.shape
    norms_per_dir: list[torch.Tensor] = []
    for k in range(K):
        v_k = directions[:, k, :, :]
        jvp_norm = directional_derivative_norm(fn, x, v_k)
        norms_per_dir.append(jvp_norm)
    stacked = torch.stack(norms_per_dir, dim=1)
    return stacked.max(dim=1).values
```

### Results / Observations

Production draft of `jacobian.py` works. Key design decision: `SpectralGapResult` uses `__slots__` dataclass — hot-path object, slots removes `__dict__` overhead.

One concern: `torch.autograd.functional.jvp` recomputes the forward pass for each direction. For $K=8$ directions, that's 8 forward passes per batch. For BERT-base on CPU, this is ~0.3s per batch of 8 examples. Acceptable for research; would need vectorization or vmap for production throughput.
