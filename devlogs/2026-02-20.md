# Dev Log — 2026-02-20: Sensitivity Functional — First Formal Definition

## What Was Built
The core SPS sensitivity functional:

```
Sens_{T,ε}(f; x) := sup_{T ∈ T, c(T,x) ≤ ε} d_Y(f(Tx), f(x)) / c(T, x)
```

Aggregated over data distribution: SPS_ε = exp(-E[Sens])

## Design Choices
- **Exponential mapping**: maps [0,∞) → (0,1]. SPS=1 is perfect invariance. Interpretable.
- **Supremum over T**: worst-case within the admissible family, not average-case. This is deliberate — we want the metric to catch the worst semantic-preserving perturbation.
- **Normalized by c(T,x)**: perturbation "cost" normalizes for severity. Larger perturbations are expected to cause more change.

## What I'm Not Sure About
The exponential feels right intuitively — there's an information-theoretic connection to entropy of the sensitivity distribution. Haven't formalized this yet.
