### Work Completed

**Drafted Definition 1 (Structured Local Sensitivity):**

$$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f_\theta;\, x) \;:=\; \sup_{\substack{T \in \mathcal{T} \\ 0 < c(T,x) \leq \varepsilon}} \frac{d_\mathcal{Y}\!\bigl(f_\theta(x),\, f_\theta(Tx)\bigr)}{c(T,x)}$$

This is the local operator-restricted Lipschitz constant: the maximum normalized output change under admissible semantic transformations within radius $\varepsilon$. The denominator $c(T, x)$ normalizes for perturbation magnitude.

Note the strict inequality $0 < c(T, x)$: the identity transform is excluded from the sup since $d_\mathcal{Y}(f(x), f(x)) / 0$ is undefined. The identity is in $\mathcal{T}$ (as the trivial "no-op" transformation) but it contributes nothing to sensitivity — by convention, $0/0$ is treated as 0.

**Drafted Definition 2 (Structured Perturbation Stability):**

$$\mathrm{SPS}_\varepsilon(f_\theta) \;:=\; \exp\!\Bigl(-\,\mathbb{E}_{x \sim \mathcal{D}}\bigl[\mathrm{Sens}_{\mathcal{T},\varepsilon}(f_\theta;\, x)\bigr]\Bigr)$$

Under the conditions that $\mathrm{Sens}$ is measurable and integrable (which I haven't formalized as axioms yet — to do), this is:
- Well-defined: $\mathrm{Sens} \geq 0$ so $\mathbb{E}[\mathrm{Sens}] \geq 0$ so $\mathrm{SPS} \in (0, 1]$
- $\mathrm{SPS} = 1$ iff $\mathbb{E}[\mathrm{Sens}] = 0$ iff the model is perfectly invariant $\mathcal{D}$-a.s.
- $\mathrm{SPS} \to 0$ as $\mathbb{E}[\mathrm{Sens}] \to \infty$ (pathologically sensitive model)

**Drafted the empirical estimator:**

$$\widehat{\mathrm{SPS}}_\varepsilon^{(n,m)}(f_\theta) \;:=\; \exp\!\left(-\frac{1}{n}\sum_{i=1}^{n} \max_{j=1,\ldots,m} \frac{d_\mathcal{Y}(f_\theta(x_i),\, f_\theta(\tau_j(x_i)))}{c(\tau_j,\, x_i)}\right)$$

where $n$ samples are drawn from $\mathcal{D}$ and $m$ transformations sampled per input. The $\max$ over $m$ approximates the $\sup$ in Definition 1.

**Prototype Python implementation:**

```python
def estimate_sensitivity(
    model: nn.Module,
    embeddings: torch.Tensor,      # (B, seq, hidden)
    transform_family: TransformationFamily,
    divergence_fn: DivergenceFn,
    epsilon: float,
    m: int,
) -> torch.Tensor:                  # (B,) — pointwise sensitivity
    fx = model(inputs_embeds=embeddings).last_hidden_state[:, 0, :]
    max_ratios = torch.zeros(embeddings.size(0))
    for _ in range(m):
        tx, cx = transform_family.sample(embeddings, epsilon)
        ftx = model(inputs_embeds=tx).last_hidden_state[:, 0, :]
        div = divergence_fn(fx, ftx)
        ratio = div / cx.clamp(min=1e-12)
        max_ratios = torch.maximum(max_ratios, ratio)
    return max_ratios
```

This is the skeleton of what will become `StructuredSensitivityEstimator.estimate()` in `src/sps/core.py`. Not production-ready yet — no proper type hints, no logging, no config object — but the logic is right.

**Design choices made today:**

1. **CLS token as representation.** For BERT-like models, $f_\theta(x)$ is the [CLS] token's hidden state at the final layer. This is the representation the task head is applied to. The sensitivity at [CLS] captures what the model actually uses for prediction.

2. **Cosine divergence as $d_\mathcal{Y}$.** Using $d_\mathcal{Y} = 1 - \cos(f(x), f(Tx))$ initially. This is bounded in $[0, 2]$ and invariant to representation magnitude. Could also use L2 distance — keeping both as options.

3. **Max over $m$ instead of mean.** Taking the max over $m$ transformations approximates the sup in Definition 1. Using the mean would give a different (lower) quantity. The max is what the definition requires.

### Results / Observations

Ran a quick sanity check with the prototype on BERT-base, SST-2 dev split (n=100 examples, m=8 transformations per example, synonym substitution family, epsilon=0.1):

| Metric | Value |
|---|---|
| Mean pointwise sensitivity | 0.341 |
| Std pointwise sensitivity | 0.187 |
| Empirical SPS | $\exp(-0.341) = 0.711$ |
| Min sensitivity (most invariant example) | 0.018 |
| Max sensitivity (most sensitive example) | 0.912 |

The distribution is right-skewed: most examples have low sensitivity, a few have very high sensitivity. This makes sense — the model is well-calibrated on most of SST-2, but there are hard examples where semantic variation causes large representation shifts.

**Cross-check:** computed the same for a randomly initialized BERT-base (untrained):

| Metric | Trained BERT | Random BERT |
|---|---|---|
| Mean sensitivity | 0.341 | 1.847 |
| Empirical SPS | 0.711 | 0.158 |

Trained model has substantially higher SPS than random initialization. This is expected and provides a sanity check: a model that has learned structured representations should be more stable to semantic-preserving perturbations than a random model.

### Lab Notes

The exponential form keeps nagging at me from an information-theoretic angle. If I write $\mathrm{SPS} = \exp(-\mathbb{E}[\mathrm{Sens}])$, this looks like the moment-generating function of $-\mathrm{Sens}$ at $t=1$, which is $M_{-\mathrm{Sens}}(1)$. The log of this is the cumulant generating function $K_{-\mathrm{Sens}}(1) = -\mathbb{E}[\mathrm{Sens}] + \frac{1}{2}\mathrm{Var}[\mathrm{Sens}] + ...$. So $\log(\mathrm{SPS}) = -\mathbb{E}[\mathrm{Sens}]$ is only the first cumulant — it's ignoring variance and higher moments.

This suggests an alternative: $\mathrm{SPS}_{\text{full}} = M_{-\mathrm{Sens}}(1)$, which would also penalize high variance of sensitivity (models that are inconsistently sensitive). But this is more complex and harder to estimate. Sticking with the mean for now.

The sanity check (trained vs random) is reassuring but also kind of vacuous. Of course a trained model is more stable than a random one. The interesting comparison is trained BERT vs trained RoBERTa vs trained DeBERTa — architecturally comparable models that differ in training procedure. That's the experiment I need to run next.

There's an instability in the max-over-m estimator: with $m=8$, the empirical sup might not be tight. A model that is sensitive in a narrow direction might look stable if none of the 8 random transformations happen to hit that direction. Need to think about how to bound the approximation error as a function of $m$.
