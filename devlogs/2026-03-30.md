**Read:** Efron & Tibshirani "An Introduction to the Bootstrap" (§6, percentile method), Bauer et al. 2020 (statistical testing in NLP), Manning et al. (Statistical NLP significance tests).

Key findings:

**Bootstrap CI for SPS:** The SPS point estimate is $\exp(-\bar{s})$ where $\bar{s}$ is the sample mean of sensitivities. The distribution of $\exp(-\bar{s})$ is non-Gaussian (it's a transformation of the sample mean). Bootstrap is the right approach — resample the sensitivity values, compute $\exp(-\bar{s})$ on each resample, take quantiles. No normality assumption required.

**Sample size guidance:** The CLT applies to $\bar{s}$ at $n \approx 30$. But we're computing $\exp(-\bar{s})$, not $\bar{s}$ directly. The CI width at $n=512$ (full SST-2 test split) with $B=200$ bootstrap resamples should be $\pm 0.01$–$\pm 0.02$. At $n=200$ (reduced for large models), $\pm 0.02$–$\pm 0.03$.

**Bauer et al.:** Report that paired bootstrap (comparing two models' scores on the same examples) has more power than unpaired bootstrap. In my setting, running both models on the same 1,821 SST-2 examples and computing a paired difference would give a tighter CI for the gap $\Delta \mathrm{SPS} = \mathrm{SPS}_{\text{RoBERTa}} - \mathrm{SPS}_{\text{BERT}}$.

**Bootstrap CI implementation:**

```python
def bootstrap_sps_ci(
    sensitivities: torch.Tensor,
    n_bootstrap: int = 200,
    alpha: float = 0.05,
    seed: int = 42,
) -> tuple[float, float]:
    """Percentile bootstrap CI for SPS = exp(-mean(sensitivities))."""
    rng = torch.Generator()
    rng.manual_seed(seed)
    n = len(sensitivities)
    estimates: list[float] = []
    for _ in range(n_bootstrap):
        idx = torch.randint(0, n, (n,), generator=rng)
        sample = sensitivities[idx]
        estimates.append(math.exp(-sample.mean().item()))
    estimates_t = torch.tensor(estimates)
    lo = float(estimates_t.quantile(alpha / 2).item())
    hi = float(estimates_t.quantile(1 - alpha / 2).item())
    return lo, hi
```

```python
class GPT2ForSPSEvaluation(nn.Module):
    """Wraps GPT-2 to expose inputs_embeds pathway for SPS evaluation."""

    def __init__(self, base_model: GPT2LMHeadModel) -> None:
        super().__init__()
        self.transformer = base_model.transformer

    def forward(
        self,
        inputs_embeds: torch.Tensor,
        attention_mask: torch.Tensor,
    ) -> torch.Tensor:
        out = self.transformer(
            inputs_embeds=inputs_embeds,
            attention_mask=attention_mask,
        )
        return out.last_hidden_state[:, -1, :]  # Last token (decoder-only)
```

The open empirical question: **can I find two models with matching accuracy but different SPS?** Current data shows accuracy and SPS both favoring RoBERTa. To demonstrate they can diverge, I need a controlled pair.

Candidate experiment: fine-tune BERT-base twice — once with aggressive data augmentation (synonym substitution during training) and once without. If augmented BERT has higher SPS but similar accuracy, that's the evidence I need. Haven't run this yet.

The scaling concern is real. GPT-2 XL at 2.2 hours per run is unusable for iterative research. The random projection approximation (project Jacobian onto $k$ random directions, use the max as the restricted norm estimate) is $O(k)$ forward passes instead of $O(K \times \text{seq\_len})$. For $k = 50$, this should be 5–10x faster. With error bounds: by Johnson-Lindenstrauss, the projection onto a random $k$-dimensional subspace preserves norms to within $\pm\varepsilon_{\text{JL}}$ where $\varepsilon_{\text{JL}} = O(\sqrt{\log(1/\delta)/k})$. For $k=50$ and $\delta=0.05$: $\varepsilon_{\text{JL}} \approx 0.3$. Too loose for precise SPS estimates but might be good enough for ordering comparisons.

The most important remaining experiment: BERT-large vs RoBERTa-base (same accuracy, different architecture). BERT-large gets ~93% on SST-2, close to RoBERTa-base's 94%. If RoBERTa's SPS advantage persists while accuracy converges, that's the paper's core empirical claim.

### Hypotheses

1. **Fine-tuning procedure affects SPS more than architecture alone.** Two BERT-base models fine-tuned with different augmentation strategies will show different SPS values while maintaining similar accuracy. Augmented BERT should have higher SPS.

2. **The spectral gap $\bar{\gamma}$ correlates with SPS across models.** If RoBERTa SPS > BERT SPS (0.793 vs 0.698), then $\bar{\gamma}_{\text{RoBERTa}} > \bar{\gamma}_{\text{BERT}}$. This is a geometric prediction from Corollary 1: higher spectral gap → better SPS lower bound.

3. **Longer pretraining consistently increases SPS.** GPT-2 (LM pretraining) < BERT (masked LM) < RoBERTa (more data + dynamic masking). The pretraining objective and data quantity both matter.

4. **Subset sampling (n=200) gives CIs of $\pm 0.015$–$\pm 0.025$.** Acceptable for model-level comparisons if the gaps are $\geq 0.05$. Should verify with a downsampling experiment on the BERT/RoBERTa data: take n=200 subsets repeatedly, check whether the ordering is preserved.
