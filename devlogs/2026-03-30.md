Bootstrap CI for SPS:

```python
def bootstrap_sps_ci(
    sensitivities: torch.Tensor,
    n_bootstrap: int = 200,
    alpha: float = 0.05,
    seed: int = 42,
) -> tuple[float, float]:
    rng = torch.Generator()
    rng.manual_seed(seed)
    import math
    estimates: list[float] = []
    n = len(sensitivities)
    for _ in range(n_bootstrap):
        idx = torch.randint(0, n, (n,), generator=rng)
        sample = sensitivities[idx]
        estimates.append(math.exp(-sample.mean().item()))
    estimates_t = torch.tensor(estimates)
    lo = float(estimates_t.quantile(alpha / 2).item())
    hi = float(estimates_t.quantile(1 - alpha / 2).item())
    return lo, hi
```

### Results / Observations

**Full results with confidence intervals (n=1,821, m=32, epsilon=0.1):**

| Model | Family | SPS | 95% CI | Mean Sens | Std Sens |
|---|---|---|---|---|---|
| bert-base-uncased | synonym | 0.698 | [0.681, 0.714] | 0.359 | 0.198 |
| bert-base-uncased | embedding | 0.731 | [0.716, 0.745] | 0.313 | 0.171 |
| roberta-base | synonym | 0.793 | [0.780, 0.805] | 0.232 | 0.147 |
| roberta-base | embedding | 0.819 | [0.808, 0.830] | 0.200 | 0.131 |

Key observation: embedding family SPS is consistently higher than synonym family SPS (BERT: 0.731 vs 0.698; RoBERTa: 0.819 vs 0.793). Embedding perturbations are continuous and scaled to exactly alpha in [0, epsilon] — they can be smaller than the minimum synonym substitution magnitude. Synonym substitutions have a larger minimum magnitude (you always swap at least one token). So embedding SPS should be higher — smaller perturbations are easier to be stable under.

**GPT-2 medium (345M params) — first attempt:**

```
RuntimeError: gpt2-medium does not support inputs_embeds with the current
  configuration. Use model.config.output_hidden_states=True.
```

GPT-2's default forward pass doesn't accept `inputs_embeds` directly through the classification head path. Need to go through the transformer trunk manually. Wrote a wrapper:

```python
class GPT2ForSPSEvaluation(nn.Module):
    def __init__(self, base_model: GPT2LMHeadModel) -> None:
        super().__init__()
        self.transformer = base_model.transformer

    def forward(self, inputs_embeds: torch.Tensor, attention_mask: torch.Tensor) -> torch.Tensor:
        out = self.transformer(inputs_embeds=inputs_embeds, attention_mask=attention_mask)
        return out.last_hidden_state[:, -1, :]  # Last token (GPT-2 is decoder-only)
```

For decoder-only models, the [CLS] token equivalent is the last token's hidden state. This is semantically less clean than [CLS] — GPT-2 doesn't have a dedicated classification token — but it's the standard approach for using GPT-2 in downstream tasks.

GPT-2 medium preliminary result (n=200 samples only, to check): SPS = 0.631 (synonym). Lower than BERT-base (0.698) and RoBERTa-base (0.793). Makes sense — GPT-2 is pretrained as a language model, not specifically optimized for semantic invariance under paraphrase.

**Scaling concern for large models:**

GPT-2 XL at 1.5B parameters: ~2.2 hours per full SPS evaluation on CPU. Infeasible for iterative experimentation. Options:
1. Random projection approximation: project the Jacobian onto a low-dimensional subspace before computing the restricted norm
2. Subset sampling: compute SPS on n=200 examples (2.5 hours → ~30 min), accept higher CI width
3. GPU (not available currently)

For this paper, will use option 2 for models > 350M params. The CI width at n=200 is approximately $\pm 0.015$ from the bootstrap results — acceptable.

### Lab Notes

The framework is in good shape. End-to-end pipeline works, results are reproducible with the seed config, confidence intervals are properly computed.

The GPT-2 wrapper for `inputs_embeds` is a signal that the transformer abstraction in `StructuredSensitivityEstimator._forward()` needs to be more flexible. The current code assumes [CLS] token at index 0. Decoder-only models need the last token. Vision transformers (if we extend to CLIP) need the [CLS] equivalent differently again. Should add a `pooling_strategy: Literal["cls", "last", "mean"]` parameter to `SPSConfig`.

The open question from the research process: **can I find two models with the same accuracy but different SPS?** The current data (BERT 91.2% / SPS 0.698 vs RoBERTa 94.1% / SPS 0.793) shows both accuracy and SPS are correlated. To show they can diverge, I need a pair where accuracy is matched. Candidates:
- Fine-tuned BERT-large vs RoBERTa-base: BERT-large gets ~93% on SST-2, close to RoBERTa-base's 94.1%
- Different fine-tuning recipes on the same base model: different LR schedules, different data augmentation during fine-tuning

The SPS vs accuracy decoupling experiment is the most important remaining piece. Everything else is already publishable.

The scaling concern is real. For CognitiveMirage / any comparison involving large LLMs, SPS computation will need GPU or approximation. The random projection approach (project Jacobian onto k=50 random directions, use max as restricted norm estimate) should be fast and approximately correct. Haven't implemented this yet.

### Hypotheses

1. **Fine-tuning procedure affects SPS more than architecture.** Two models with the same architecture (BERT-base) fine-tuned with different LR schedules or augmentation strategies will show different SPS values while maintaining similar accuracy. The pretraining gives the base representations; fine-tuning determines how sensitivity is organized relative to the semantic manifold.

2. **Longer pretraining consistently increases SPS.** RoBERTa was pretrained for longer on more data than BERT. GPT-2 was pretrained as a language model, not explicitly for semantic invariance. If pretraining duration correlates with SPS, BERT-large (more params, longer training) should have higher SPS than BERT-base. Hypothesis: SPS(BERT-large) > SPS(BERT-base).

3. **The spectral gap $\bar{\gamma}$ will correlate with SPS across models.** If RoBERTa has higher SPS than BERT (0.793 vs 0.698), it should also have a higher spectral gap (its Jacobian top singular vectors are more orthogonal to semantic directions). This is a geometric prediction from the theory that I haven't verified empirically yet.
