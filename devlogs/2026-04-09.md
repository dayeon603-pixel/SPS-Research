**Delta-method 95% CI for rSPS (metrics.py + estimate_sps.py):**

rSPS = SPS(T_emb) / SPS(T_arb) = exp(−mean_s_emb) / exp(−mean_s_arb) = exp(mean_s_arb − mean_s_emb).

Delta method: log(rSPS) = mean_s_arb − mean_s_emb, so Var(log rSPS) ≈ std_s_emb²/n_emb + std_s_arb²/n_arb (independence). Back-transform via exp(log_rSPS ± 1.96·SE).

Changes:
- `estimate_arbitrary_sps()` in metrics.py: added `return_full: bool = False` parameter. When True, returns the full estimator result dict (sps, mean_sensitivity, std_sensitivity, n_samples) instead of just the float. Default False preserves backward compatibility; no tests use this function directly.
- `delta_method_rsps_ci()` in estimate_sps.py: new function, 34 lines. Takes emb_{mean,std,n} and arb_{mean,std,n}, returns (point, lo, hi). Uses hardcoded z-scores (1.96 for 95%, 2.576 for 99%) — no scipy dependency.
- `main()` in estimate_sps.py: calls `estimate_arbitrary_sps(..., return_full=True)`, computes rsps_ci, prints "95% CI [lo, hi] (delta method, log-space)" and a line indicating whether the CI excludes 1 (statistically significant departure from generic smoothness).

**Syntax verification:** `python3 -m py_compile` passes on both modified files.

### Work Completed

**adversarial_worst_direction() — jacobian.py:** Constructive dual of `restricted_operator_norm`. Returns `argmax_{v ∈ A_x} ||Jf(x) v||` — the semantic direction achieving maximum JVP norm. Implemented as argmax over K JVP evaluations (same O(K) cost as restricted_operator_norm).

**AdversarialEmbeddingFamily (T_adv) — transformations.py:** New transformation family. Delegates direction construction to T_emb (same synonym/random A_x), but picks v* = worst-case direction via JVP before perturbing. The ratio SPS(T_adv)/SPS(T_emb) is the key new metric — quantifies how near-adversarial random synonym sampling is.

**estimate_sps.py integration:**
- Imports AdversarialEmbeddingConfig, AdversarialEmbeddingFamily
- New `--skip-adversarial` flag (default: run)
- Adversarial SPS analysis block: builds T_adv, runs estimator on all batches, reports SPS(T_adv) and rSPS_adv = SPS(T_adv)/SPS(T_arb)
- Results table: T_adv row inserted between T_emb and T_arb
- Interpretation: ratio <1.1 → random directions already near-adversarial; >2× → large adversarial gap

**Tests:** 4 new tests for AdversarialEmbeddingFamily (output shapes, epsilon constraint, direction delegation, non-negativity). 33/33 tests passing.

**API exports:** adversarial_worst_direction, AdversarialEmbeddingConfig, AdversarialEmbeddingFamily added to src/sps/__init__.py.

### Results / Observations

The adversarial direction construction is O(K) JVP evaluations (same as restricted_operator_norm) — no extra cost beyond what spectral gap already computes. The key question this answers: is SPS(T_emb) a good proxy for the worst-case semantic sensitivity? If SPS(T_adv)/SPS(T_emb) ≈ 1, then yes; if large, random synonym sampling understates vulnerability.

Cannot run on model without transformers/GPU in this environment. Expected result based on spectral gap theory: with K=8 random directions and d=768 embedding space, the random cover is sparse, so the adversarial gap is likely 2–5×.

### Hypotheses

SPS(T_adv)/SPS(T_emb) should be related to sqrt(d/K) by a concentration argument — for d=768 and K=8, this gives ~sqrt(96) ≈ 10×. However, synonym directions are not random in the embedding space; they live in a low-dimensional semantic subspace. If the semantic subspace has effective dimension k << d, then the adversarial gap may be much smaller: sqrt(k/K) with k~50 gives ~2.5×.
