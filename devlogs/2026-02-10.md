**Experiment: Gaussian noise injection into [CLS] embedding layer output.**

```python
def add_gaussian_noise(
    embeddings: torch.Tensor,
    sigma: float,
    seed: int = 42,
) -> torch.Tensor:
    torch.manual_seed(seed)
    return embeddings + sigma * torch.randn_like(embeddings)
```

Ran this at sigma values: 0.01, 0.05, 0.1, 0.25, 0.5. For each sigma, computed prediction consistency over the 872 examples: what fraction of perturbed examples match the original prediction.

```python
consistency = (perturbed_preds == original_preds).float().mean().item()
```

Also computed cosine similarity between original [CLS] representation and perturbed [CLS] representation at each sigma level.

**What I noticed when I looked more carefully:** The predictions that did change were almost entirely on borderline examples — examples where the original confidence was near 50%. The examples that flipped were the ones sitting right on the decision boundary.

### Lab Notes

Spent about 30 minutes feeling good about BERT's robustness before something felt off. The CLS embedding at sigma=0.25 has cosine similarity 0.78 with the original — that's a meaningfully different vector. But predictions only changed 5.7% of the time. Those numbers don't cohere unless the prediction layer is doing something very insensitive to small representation shifts.

And it is. At high confidence, the pre-softmax logits are like +4.2, −4.2 for a two-class problem. You can push those logits around by ±1 and the prediction doesn't change. That's just arithmetic, not robustness.

Also noticed: the perturbation I added is noise in all 768 directions simultaneously. It disrupts syntactic structure, morphological features, positional encoding contributions, everything at once. Even if the model were fragile to specifically semantic perturbations, this test would hide it. Random noise in high-dimensional space is, on average, orthogonal to any particular semantic direction. It's the worst possible probe for semantic robustness.

Wrote in notes: *"measuring the wrong layer. the stability is in the softmax, not the encoder."*

### Hypotheses

1. **Softmax saturation explains the consistency numbers.** Models trained until convergence on classification tasks develop high-confidence predictions on clean data. Softmax at high confidence is nearly flat. This makes any metric based on prediction consistency almost useless for measuring representation quality.

2. **Random perturbations are semantically inert in high-dimensional space.** A 768-dimensional Gaussian vector is, with overwhelming probability, nearly orthogonal to any specific semantic direction. So even large-sigma noise may not align with the semantically relevant axes.

3. **Useful robustness measurements need to be grounded in the semantic manifold.** To ask whether a model is invariant to meaning-preserving changes, the perturbations must actually preserve meaning. Random noise doesn't satisfy this. The perturbation needs semantic structure.
