## [08:30 UTC] Reading Session — Adversarial Robustness Literature

### State at Session Start

Project is brand new. No commits, no framework. Goal today: survey existing work on robustness metrics for transformers before writing any code. Starting with AdvGLUE, CheckList, and Jacovi & Goldberg 2020 to understand the landscape.

### Work Completed

Read three papers:

**Jacovi & Goldberg 2020 ("Towards Faithfully Interpretable NLP Systems"):** Key claim is that accuracy is a poor proxy for "understanding" — a model can achieve high accuracy through spurious correlations that are invisible to output-based probing. The faithfulness vs. plausibility distinction maps almost directly onto what I want to measure: plausible explanations are coherent post-hoc stories, faithful ones reflect what the model actually computed. Output-only robustness metrics are plausible-but-not-necessarily-faithful.

**Wang et al. 2021 (AdvGLUE):** Adversarial robustness benchmark. They apply worst-case textual attacks (word substitution, syntactic perturbation) and measure accuracy drop. The key issue for my purposes: AdvGLUE optimizes for attacks. It asks "what is the worst semantic transformation that breaks the model?" I want neutral characterization — "how sensitive is the model across the distribution of semantics-preserving transformations?" Different question.

**Ribeiro et al. 2020 (CheckList):** Behavioral testing with capability matrices. Also accuracy-driven. Well-designed but same issue: the metric is prediction flip rate, which inherits all the softmax saturation problems I already know about.

Noted a pattern: every paper in the adversarial robustness space measures either (a) prediction flip rate, or (b) accuracy drop. Neither measures representation-level sensitivity. The field is leaving the internal state of the model entirely uncharacterized.

### Results / Observations

Literature gap identified: no existing metric (as of the papers I've read) captures the sensitivity of transformer representations to structure-preserving perturbations at the operator level. All existing metrics operate in output space or classification label space.

### Hypotheses

1. Output-space metrics conflate softmax saturation with representation quality — this needs empirical confirmation.
2. Adversarial benchmarks are not the right tool for neutral characterization of semantic stability.
3. There may be a clean formal connection between representation-level sensitivity and the Jacobian of the model — worth exploring once the empirical issue is nailed down.

### Next Steps

- Design Gaussian noise injection experiment: explicitly test whether output stability under noise measures softmax saturation vs. genuine robustness
- Define what "semantic preservation" should mean operationally
- Read: Dong et al. 2021 (attention robustness) and Ravichander et al. 2020 (probing benchmarks)

## [14:22 UTC] Implementation Session — Gaussian Noise Injection

### State at Session Start

Literature survey done. No code written yet. Setting up the first experiment: inject Gaussian noise into BERT embedding outputs, measure prediction consistency. Hypothesis going in: if outputs are stable under noise, is that actually evidence of semantic robustness? Expecting to find it isn't.

Setup:
- `bert-base-uncased` (110M params, 12 layers, $d = 768$ hidden)
- Dataset: SST-2 dev split (872 examples)
- Device: CPU (no GPU available today)

### Work Completed

**Experiment: Gaussian noise injection into embedding layer output.**

```python
def add_gaussian_noise(
    embeddings: torch.Tensor,
    sigma: float,
    seed: int = 42,
) -> torch.Tensor:
    torch.manual_seed(seed)
    return embeddings + sigma * torch.randn_like(embeddings)
```

Ran this at $\sigma \in \{0.01, 0.05, 0.10, 0.25, 0.50\}$. For each $\sigma$, computed prediction consistency over 872 examples: fraction of perturbed examples where $\hat{y}(x + \eta) = \hat{y}(x)$.

```python
consistency = (perturbed_preds == original_preds).float().mean().item()
```

Also computed cosine similarity between original and perturbed [CLS] representation at each $\sigma$.

Looked at which examples flipped:
```python
flip_mask = (perturbed_preds != original_preds)
flip_confidences = original_probs[flip_mask].max(dim=1).values
# mean confidence of flipped examples: 0.517
# mean confidence of stable examples: 0.893
```

### Results / Observations

| $\sigma$ | Prediction Consistency | CLS Cosine Similarity |
|---|---|---|
| 0.01 | 0.998 | 0.9991 |
| 0.05 | 0.991 | 0.9816 |
| 0.10 | 0.978 | 0.9327 |
| 0.25 | 0.943 | 0.7801 |
| 0.50 | 0.887 | 0.5532 |

At $\sigma = 0.25$, cosine similarity has dropped to 0.78 — embeddings are genuinely different vectors — but prediction consistency is still 94.3%. The flips are almost entirely borderline examples (mean confidence of flipped examples: 0.517; stable: 0.893).

This is the central problem: at high confidence, BERT's pre-softmax logits are roughly $\pm 4.2$ for a two-class problem. You can shift those logits by $\pm 1.0$ and the prediction doesn't change. That's arithmetic, not robustness.

The perturbation is also semantically incoherent: I'm adding noise in all 768 directions simultaneously, disrupting syntactic structure, morphological features, and positional encoding contributions all at once. A random vector in $\mathbb{R}^{768}$ is with overwhelming probability nearly orthogonal to any specific semantic direction — this is a concentration of measure argument. So even large-$\sigma$ noise is not testing semantic robustness at all.

Wrote in notes: *"measuring the wrong layer. the stability is in the softmax, not the encoder."*

### Hypotheses

1. **Softmax saturation explains the consistency numbers.** High-confidence models at evaluation time have essentially flat softmax. Any metric based on prediction consistency conflates representation quality with softmax geometry.

2. **Random perturbations are semantically inert in high-dimensional space.** By concentration of measure, a random direction in $\mathbb{R}^{768}$ has negligible projection onto any specific semantic direction. Adding isotropic noise is not probing semantic invariance.

3. **Useful robustness measurements need to be grounded in the semantic manifold.** The perturbation family $\mathcal{T}$ must be restricted to transformations that actually preserve semantic content — otherwise the stability measurement is uninterpretable.

### Next Steps

- Design output-space consistency metric over actual paraphrases (not random noise)
- Define what "semantic preservation" means operationally — something testable without circularity
- Sketch a constrained transformation family $\mathcal{T}$ formally

## [20:15 UTC] Whiteboard Session — What Does Semantic Preservation Mean?

### State at Session Start

Gaussian noise experiment completed and failure logged. Central open question: how do I define "semantic preservation" without circularity? If I use the model itself to define what's semantically equivalent, the metric is trivially circular. If I use human annotation, it's expensive and doesn't scale.

### Work Completed

No code. Two hours of whiteboard work on the formal definition problem.

**The circularity problem stated precisely:** I want to measure whether $f_\theta$ is invariant under semantics-preserving transformations. But to know which transformations are semantics-preserving, I need to know what $f_\theta$ (or something like it) considers equivalent. Using $f_\theta$ to define $\mathcal{T}$ makes the metric tautologically 1.

**Three candidate approaches:**

*Option A — NLI-based filtering:* Use an NLI model to check that $\text{entailment}(x, Tx) \wedge \text{entailment}(Tx, x)$. Reject $T$ if NLI model says they're not equivalent.
- Problem: circular if the NLI model is in the same family as the model being tested. Also, NLI models have their own robustness problems.

*Option B — Human-curated families:* Define $\mathcal{T}$ as synonym substitution, paraphrase templates, back-translation. Humans guarantee semantic preservation by construction.
- Problem: coverage is incomplete. Also, "synonym" is context-dependent — "fast" and "quick" are synonyms for speed but not for food ("fast food" vs. "quick food").

*Option C — Operational definition with explicit scope:* $\mathcal{T}$ is a curated finite family of transformation types, each validated to preserve the ground-truth label on the evaluation corpus. Semantic preservation is defined operationally as label preservation, not meaning preservation in the philosophical sense.
- Less elegant but tractable. Honest about what it's measuring.

Decision: go with Option C. Define $\mathcal{T}$ operationally. The framework should be explicit that it measures label-preserving sensitivity, not some deeper notion of "meaning."

**Sketched the formal definition:**

$$\mathcal{T} = \{T : \mathcal{X} \to \mathcal{X} \mid T \text{ preserves ground-truth label on } \mathrm{supp}(\mathcal{D})\}$$

This immediately raises the question: how fine-grained should the transformation magnitude $c(T, x)$ be? If I just count how many tokens changed, I'm treating all token changes as equal. But swapping "dog" for "canine" is a smaller semantic step than swapping "good" for "decent."

The magnitude should live in embedding space, not token space:

$$c(T, x) = \|Tx - x\|_2 \quad \text{where } x, Tx \in \mathbb{R}^{d \times \text{seq}}$$

### Results / Observations

No quantitative results. Design decision documented: operational definition of $\mathcal{T}$ via label preservation. Magnitude $c(T, x)$ in embedding space, not token space.

The key insight from this session: by choosing $c(T, x) = \|Tx - x\|_2$ in embedding space, the sensitivity functional

$$\mathrm{Sens}_{\mathcal{T},\varepsilon}(f_\theta; x) = \sup_{\substack{T \in \mathcal{T} \\ 0 < c(T,x) \leq \varepsilon}} \frac{d_\mathcal{Y}(f_\theta(x), f_\theta(Tx))}{c(T, x)}$$

is the local Lipschitz constant of $f_\theta$ restricted to the orbit of $x$ under $\mathcal{T}$. That's a clean characterization.

### Hypotheses

1. Operational definition (label preservation) is good enough to build something meaningful — philosophical purity on "semantic equivalence" can come later.
2. The sensitivity functional in embedding space will have a clean Jacobian characterization in the $\varepsilon \to 0$ limit.
3. The key challenge will be making $\mathcal{T}$ concrete and computable — what are the actual transformation families?

### Next Steps

- Implement output-space consistency metric over actual paraphrase pairs (next session)
- Begin sketching axioms for a formal $\mathcal{T}$ — what properties must it satisfy?
- Think about the $\varepsilon \to 0$ limit and whether a Jacobian characterization falls out naturally
