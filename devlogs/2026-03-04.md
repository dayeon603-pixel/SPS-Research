### Work Completed

No code. Design decisions:

**Q1: Should `StructuredSensitivityEstimator` know about the model architecture?**

Option A: Pass a `ForwardFn = Callable[[torch.Tensor], torch.Tensor]` ŌĆö model-agnostic. The caller wraps their model.
Option B: Accept an `nn.Module` and handle [CLS] extraction internally.

Decision: Option B. The estimator needs to run $m$ forward passes per batch. If the caller passes a generic callable, they can't benefit from batching, KV-caching, etc. Taking `nn.Module` lets the estimator handle the batching logic and the CLS extraction internally. Trade-off: slightly less flexible, but much easier to use correctly.

**Q2: Where does embedding extraction live?**

The transformation family operates in embedding space (it adds $\alpha v$ to the embedding layer output). So the estimator needs to run the model from embeddings, not from input_ids. This means:
- Extract embeddings from input_ids once (via `model.get_input_embeddings()`)
- Run `m` forward passes with `inputs_embeds=` keyword
- This bypasses the embedding lookup for the perturbed passes ŌĆö correct, since the perturbation is in embedding space

Embedding extraction lives in `SPSEstimator._get_embeddings()` ŌĆö called once per batch, not per transformation.

**Q3: Pydantic v2 for config?**

Yes. `SPSConfig` as a `BaseModel` with `field_validator` for device. Config passed to both `StructuredSensitivityEstimator` and `SPSEstimator` ŌĆö single source of truth. No mutable config state after construction.

**Q4: Which divergence functions to support?**

`"cosine"`: $d(a, b) = 1 - \cos(a, b) \in [0, 2]$. Invariant to representation norm. Default.
`"l2"`: $d(a, b) = \|a - b\|_2$. Scale-sensitive but directly interpretable in embedding space.
`"kl"`: KL divergence (for probability outputs). Needed for output-space SPS (not representation-space).

Implementing as `DivergenceType = Literal["cosine", "l2", "kl"]` with `get_divergence_fn(type)` factory.

**Q5: Transformation family base class?**

Abstract base class `TransformationFamily(ABC)`:
- `sample(embeddings, input_ids, epsilon) -> (perturbed, magnitudes)` ŌĆö required
- `semantic_directions(embeddings, input_ids) -> (B, K, seq, hidden)` ŌĆö optional (needed for JVP)

Three concrete families:
- `EmbeddingPerturbationFamily` ŌĆö continuous, synonym-derived directions
- `SynonymSubstitutionFamily` ŌĆö discrete token replacement (not differentiable)
- `ArbitraryEmbeddingFamily` ŌĆö random L2-ball perturbations (for rSPS baseline)

`AdversarialEmbeddingFamily` is built on top of `EmbeddingPerturbationFamily` with worst-case direction selection.

The `StructuredSensitivityEstimator._forward()` fallback chain for HuggingFace models is inelegant but necessary ŌĆö different architectures expose `last_hidden_state` differently. ## [21:00 UTC] Whiteboard Session ŌĆö Proposition 1 Proof + A4

### State at Session Start

Implementation done. Quick theory session to write Proposition 1 formally before sleeping.

### Work Completed

**Proposition 1 (Boundedness):** $0 < \mathrm{SPS}_\varepsilon(f_\theta) \leq 1$.

Upper bound ($\leq 1$): $\mathrm{Sens} \geq 0$ everywhere (since $d_\mathcal{Y} \geq 0$ and $c > 0$). Therefore $\mathbb{E}[\mathrm{Sens}] \geq 0$. Applying $\exp(-\cdot)$: $\mathrm{SPS} = \exp(-\mathbb{E}[\mathrm{Sens}]) \leq \exp(0) = 1$. No extra assumption needed.

Lower bound ($> 0$): I wrote $\exp(-\mathbb{E}[\mathrm{Sens}]) > 0$ because exp is always positive. Then tried to prove it.

**Wait ŌĆö when is $\exp(-x) > 0$?** Always, for any finite $x$. But if $x = +\infty$, then $\exp(-\infty) = 0$. So the claim $\mathrm{SPS} > 0$ requires $\mathbb{E}[\mathrm{Sens}] < +\infty$.

Is $\mathbb{E}[\mathrm{Sens}]$ finite? Not necessarily without an assumption. If there exists a positive-$\mathcal{D}$-probability set of $x$ where $\mathrm{Sens}_{\mathcal{T},\varepsilon}(f_\theta; x) = +\infty$, then the expectation diverges and $\mathrm{SPS} = 0$.

For a locally Lipschitz $f_\theta$ and bounded $\varepsilon$, $\mathrm{Sens}$ is finite everywhere. But I don't want to assume global Lipschitz (that's Theorem 3's hypothesis). So I need an explicit integrability assumption.

**Added A4 (Integrability):**

$$\mathbb{E}_{x \sim \mathcal{D}}[\mathrm{Sens}_{\mathcal{T},\varepsilon}(f_\theta; x)] < \infty.$$

With A4, the proof is: $\exp(-\mathbb{E}[\mathrm{Sens}]) > 0$ because $\mathbb{E}[\mathrm{Sens}] < \infty$ (A4) implies the exponent is finite, and $e^{-t} > 0$ for all finite $t$.

Without A4, the proof fails. The correct claim without A4 is only $\mathrm{SPS} \geq 0$.

Updated `theory/definitions.md` with the full A1ŌĆōA5 list, including A4. Updated `theory/proofs.md` with the corrected Proposition 1 proof with explicit "by A4" annotation.

### Results / Observations

Proposition 1 proof written and correct. A4 is the key integrability assumption ŌĆö without it, $\mathrm{SPS} = 0$ is possible, which would be a degenerate/meaningless case.
