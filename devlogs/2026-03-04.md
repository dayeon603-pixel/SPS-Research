# Dev Log — 2026-03-04: Core Implementation — sps/core.py + sps/jacobian.py

## What Was Built
- `src/sps/core.py`: SPS computation engine
  - `compute_sps()`: main entry point
  - `sensitivity()`: per-example sensitivity computation
  - `aggregate_sps()`: exponential aggregation over dataset
- `src/sps/jacobian.py`: Jacobian-vector product computation
  - Uses `torch.func.jvp` for efficient forward-mode AD
  - Batched computation for transformer models
  - Restricted operator norm estimation via power iteration

## Also Built
- `src/sps/transformations.py`: Transformation families
  - `SynonymSubstitution`: WordNet-based synonym swaps
  - `BackTranslation`: roundtrip translation via MarianMT
  - `SyntaxTransform`: dependency tree manipulation via spaCy
- `src/sps/metrics.py`: comparison metrics (BERTScore, cosine similarity baselines)

## Tests
- `tests/test_core.py`: SPS computation correctness
- `tests/test_jacobian.py`: JVP shape, gradient flow
- `tests/test_transformations.py`: transformation family properties

## Status
Core framework is functional. Can compute SPS for any PyTorch model + transformation family. Need to validate on real models now.
