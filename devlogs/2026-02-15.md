# Dev Log — 2026-02-15: The Key Realization — Constrained Transformation Families

## Context
Two weeks of failed experiments (random noise, output-only metrics). Both masked the failure modes I wanted to detect.

## The Insight
Wrote in notes: *"not all perturbations are equal — only structure-preserving ones should count"*

Random noise and arbitrary transformations are semantically destructive. If I swap "dog" for "photosynthesis", the model SHOULD be sensitive. The question is whether it's sensitive when I swap "dog" for "canine."

## What This Means
The framework needs a **constrained transformation family T** — only transformations that preserve semantic content participate in the stability measurement.

## Open Problem
How to define "semantic preservation" formally without circularity? Three options:
1. Axiomatic (elegant but hard to verify)
2. Operational — curated families: synonym substitution, controlled paraphrase, back-translation
3. NLI-based (circular — using a model to evaluate a model)

Going with option 2 for now. Less elegant but tractable.
